1. Comparativo entre as diferentes versões do HTML:

  • HTML 1.0 (1991): Foi a primeira versão do HTML, bastante básica e destinada a estruturar documentos simples para a web.

  • HTML 2.0 (1995): Introduziu novos elementos e atributos, incluindo formulários.

  • HTML 3.2 (1997): Introduziu muitos dos elementos e atributos que ainda usamos hoje, como tabelas e applets.

  • HTML 4.0 (1997-1999): Foi uma grande evolução, introduzindo CSS e JavaScript mais sofisticado. Dividiu o HTML em dois grupos: HTML estrito (strict) e HTML transitório (transitional).

  • XHTML (2000-2008): Uma versão reformulada do HTML 4, seguindo as regras do XML. Embora fosse uma evolução conceitual, não ganhou a mesma popularidade que o HTML.

  • HTML5 (2014): A versão mais recente, trazendo uma série de novos recursos, semântica melhorada e capacidades multimídia integradas. Introduziu elementos semânticos como <header>, <section>, <article>, <nav>, <aside>, <footer></footer> , que ajudam a definir a estrutura de uma página de forma mais significativa.

2. Web Semântica:

A Web Semântica é uma extensão da World Wide Web que visa tornar o conteúdo da internet mais compreensível por máquinas. Ela se concentra em tornar as informações acessíveis e interpretáveis por computadores, permitindo que eles entendam o significado dos dados. Isso facilita a busca e a organização de informações, bem como o desenvolvimento de serviços e aplicativos mais avançados.

3. Protocolos HTTP e HTTPS:

  • HTTP (Hypertext Transfer Protocol): É um protocolo de comunicação usado na World Wide Web para a transferência de dados entre um navegador web e um servidor web. Os dados são transmitidos de forma não criptografada, o que significa que podem ser interceptados por terceiros.

  • HTTPS (Hypertext Transfer Protocol Secure): É uma versão segura do HTTP. Utiliza um protocolo de criptografia (SSL/TLS) para proteger os dados durante a transferência. Isso significa que os dados são criptografados e muito mais difíceis de serem interceptados por terceiros mal-intencionados.

4. URL (Uniform Resource Locator):

Uma URL é o endereço de um recurso na internet. Ela define o local específico onde um recurso pode ser encontrado na web. As URLs consistem em diferentes partes, incluindo:

  • Esquema (Scheme): Indica o protocolo utilizado (por exemplo, http, https, ftp).

  • Host: Indica o servidor onde o recurso está localizado.

  • Caminho (Path): Especifica a localização do recurso no servidor.

  • Parâmetros e Fragmento: Opcionalmente, pode conter informações adicionais para serem passadas ao servidor.

Exemplo de uma URL: https://www.exemplo.com/pagina1
